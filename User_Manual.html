1. Introduction

1.1 About SParking

SParking is a comprehensive smart parking management system designed to streamline parking operations for facilities of all sizes. From small parking lots to large multi-level commercial parking structures, SParking provides the tools you need to efficiently manage vehicles, process payments, and gain valuable insights into your parking operations.

Key Features:

  • Automatic Number Plate Recognition (ANPR) - Automated vehicle identification using AI-powered camera systems
  • Real-time Monitoring - Live dashboard with occupancy tracking and camera feeds
  • Multi-Payment Support - Accept Cash, UPI, Card, and digital wallet payments
  • Zone-based Management - Organize parking by zones, levels, and vehicle types
  • Comprehensive Analytics - Detailed reports and predictive analytics
  • Hardware Integration - Control gates, displays, and ticket printers
  • Multi-currency Support - INR, USD, EUR, GBP

1.2 System Requirements

Component Requirement
Web Browser Google Chrome (recommended), Firefox, Safari, or Edge (latest versions)
Internet Connection Stable broadband connection (minimum 5 Mbps for camera streaming)
Screen Resolution Minimum 1280x720, recommended 1920x1080 or higher
Device Desktop, laptop, or tablet (responsive design supported)

1.3 Getting Started

Logging In

  1. Open your web browser and navigate to your SParking URL (provided by your administrator)
  2. Enter your Email Address in the email field
  3. Enter your Password in the password field
  4. Click the "Sign In" button
  5. Upon successful login, you will be redirected to the Dashboard
Note: If you forget your password, contact your system administrator to reset it. For security reasons, passwords must be at least 8 characters and include uppercase, lowercase, numbers, and special characters.

First-Time Setup Checklist

Step Action Location
1 Configure organization settings Settings > Organization
2 Set up parking lot details Parking Lots > Add New
3 Create zones Zones > Add Zone
4 Configure parking slots Slots > Add Slots
5 Connect cameras Cameras > Add Camera
6 Set up payment methods Settings > Payments

3. Parking Management

3.1 Parking Lots

Parking lots are the top-level organizational unit in SParking. Each lot represents a physical parking facility.

Creating a New Parking Lot

  1. Navigate to Parking Lots from the sidebar
  2. Click the "Add Parking Lot" button
  3. Fill in the required information:
    • Name: Display name for the parking lot
    • Address: Physical street address
    • City: City location
    • Country: Select from dropdown
    • Currency: Default currency for pricing (INR, USD, EUR, GBP)
    • Venue Type: Mall, Airport, Hospital, Hotel, Commercial, Stadium, Cinema, or Residential
  4. Enable optional features:
    • EV Charging: If electric vehicle charging is available
    • Valet Service: If valet parking is offered
  5. Click "Create" to save

Parking Lot Status

Status Description When to Use
ACTIVE Fully operational Normal operations
MAINTENANCE Under maintenance During repairs or upgrades
CLOSED Not accepting vehicles After hours or permanent closure

3.2 Zones

Zones help organize your parking lot into logical sections. Each zone can have different characteristics and pricing.

Zone Types

Type Code Description
General GENERAL Standard parking for all vehicles
VIP VIP Premium parking spots (may have higher rates)
EV Charging EV_CHARGING Dedicated spots with charging stations
Accessible DISABLED Handicapped accessible parking
Staff STAFF Reserved for employees
Visitor VISITOR Guest parking areas
Short Term SHORT_TERM Quick parking (15-30 min)
Long Term LONG_TERM Extended parking (airport style)

Valet VALET Valet service parking

Creating a Zone

  1. Navigate to Zones from the sidebar
  2. Click "Add Zone"
  3. Enter zone details:
    • Name: e.g., "Ground Floor A"
    • Code: Short code e.g., "GF-A"
    • Level: Floor number (0 for ground)
    • Type: Select from zone types
    • Color: Pick a color for visual identification on maps
    • Description: Optional notes
  4. Click "Create"

3.3 Slots

Slots represent individual parking spaces within zones.

Bulk Slot Creation

  1. Navigate to Slots from the sidebar
  2. Click "Add Slots"
  3. Select the Zone where slots will be added
  4. Configure slot generation:
    • Prefix: e.g., "A" (creates A001, A002, etc.)
    • Start Number: Starting number
    • Count: Number of slots to create
    • Slot Type: Standard, Compact, Large, Handicapped, EV Charging, Motorcycle, VIP
    • Vehicle Type: Allowed vehicle type
  5. Enable features as needed:
    • EV Charger: Slot has charging capability
    • Accessible: Wheelchair accessible
  6. Click "Create Slots"

Slot Status Indicators

Status Color Description
Available Green Empty and ready for use
Occupied Red Currently has a vehicle
Reserved Blue Booked/pre-reserved
Maintenance Orange Under maintenance
Blocked Gray Temporarily unavailable

3.4 Cameras

Cameras are used for ANPR (Automatic Number Plate Recognition) and live monitoring.

Adding a Camera

  1. Navigate to Cameras from the sidebar
  2. Click "Add Camera"
  3. Enter camera details:
    • Name: Descriptive name e.g., "Entry Gate Camera 1"
    • RTSP URL: Camera stream URL (e.g., rtsp://192.168.1.100:554/stream)
    • ONVIF URL: For PTZ control (optional)
    • Username: Camera login username
    • Password: Camera login password
    • Position: Description of camera location
    • Coverage: Number of slots this camera monitors
  4. Enable features:
    • Infrared: Night vision capability
    • PTZ: Pan-Tilt-Zoom support
  5. Assign camera to slots for ANPR detection
  6. Click "Save"
Tip: For optimal ANPR performance, position cameras at a 15-30 degree angle to capture license plates clearly. Ensure adequate lighting or use IR-enabled cameras for 24/7 operation.

4. Daily Operations

4.1 Token Management

Tokens are the core of parking operations - they track vehicle entry, parking duration, and fees.

Token Types

Type Description Use Case
QR Code Generated QR code ticket Standard parking (printed or digital)
RFID Radio-frequency tag Monthly pass holders
Barcode Barcode ticket Legacy ticket systems
ANPR License plate recognition Ticketless entry/exit
Manual Manually entered Backup when systems are down

Token Status Flow

ACTIVE → (Vehicle exits and pays) → COMPLETED

ACTIVE → (Ticket lost) → LOST → (Lost fee paid) → COMPLETED

ACTIVE → (Time exceeded) → EXPIRED

ACTIVE → (Cancelled) → CANCELLED

Creating a Manual Token

  1. Navigate to Tokens
  2. Click "Generate Token"
  3. Enter vehicle details:
    • License Plate: Vehicle registration number
    • Vehicle Type: Car, SUV, Motorcycle, etc.
    • Entry Type: Select token type
  4. System automatically:
    • Assigns a token number
    • Records entry timestamp
    • Allocates an available slot
  5. Print ticket if using physical tokens

Processing Token Exit

  1. Search for the token by:
    • Token number
    • License plate
    • QR code scan
  2. Review parking details:
    • Entry time
    • Duration
    • Calculated fee
  3. Select payment method: Cash, UPI, or Card
  4. Process payment
  5. Token is marked as COMPLETED
  6. Gate opens automatically

  5. Click "Save"

4.4 Vehicle Entry Process

The standard vehicle entry workflow:

  1. Vehicle Arrives: Vehicle approaches entry gate
  2. Detection:
    • ANPR Mode: Camera captures license plate automatically
    • Manual Mode: Operator enters plate number or issues ticket
  3. Vehicle Check: System checks if vehicle is blacklisted
    • If blacklisted: Gate remains closed, alert generated
    • If whitelisted/VIP: Gate opens automatically
    • If regular: Proceed to token generation
  4. Slot Allocation: System assigns available slot based on vehicle type
  5. Token Generation: Entry token created with timestamp
  6. Gate Opens: Entry barrier lifts
  7. Display Update: Digital signage shows available slots count
  8. Vehicle Parks: Driver proceeds to allocated area

4.5 Vehicle Exit Process

The standard vehicle exit workflow:

  1. Vehicle Approaches Exit: Vehicle arrives at exit gate
  2. Token Identification:
    • ANPR Mode: Plate recognized, token retrieved automatically
    • QR/Barcode: Driver scans ticket
    • Manual: Operator searches token
  3. Fee Calculation:
    • System calculates duration
    • Applies pricing rules
    • Adds any applicable taxes
  4. Payment:
    • Cash: Operator collects cash and marks paid
    • UPI: Customer scans QR and pays digitally
    • Card: Card payment processed
    • Wallet: Deducted from digital wallet
  5. Token Completed: Token marked as COMPLETED
  6. Gate Opens: Exit barrier lifts
  7. Slot Released: Parking slot marked as available
  8. Receipt: Optional receipt generated

Adding a Bank Account

8. Troubleshooting

Common Issues and Solutions

Issue Possible Cause Solution
Cannot log in Wrong credentials or account locked Reset password or contact admin
Camera shows offline Network issue or camera power Check network cable, restart camera
Gate not opening Hardware connection issue Check controller connection, manual override
ANPR not detecting plates Poor lighting or camera angle Adjust camera, clean lens, check lighting
Payment failed Gateway issue or insufficient funds Check payment gateway status, try different method
Slow performance Network congestion or browser cache Clear browser cache, check internet speed
Reports not generating Large date range or system load Try smaller date range, wait and retry
Display not updating Connection to display lost Check display connection, restart

Emergency Procedures

System Down

  1. Switch to manual ticket processing
  2. Use physical barrier controls
  3. Record entries/exits manually
  4. Contact technical support immediately
  5. Enter manual records into system once restored

Payment Gateway Down

  1. Accept cash payments only
  2. Issue manual receipts
  3. Record all transactions for later entry
  4. Check gateway status page
  5. Reconcile once restored

Power Outage

  1. UPS should maintain critical systems briefly
  2. Use manual gate overrides
  3. Flashlights for ticket verification
  4. System auto-recovers when power restored
